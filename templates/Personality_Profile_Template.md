# Personality Profile — Template

## Purpose

A Personality Profile maps the behavioral patterns, communication style, and relational architecture of an agent. Where the Soul File captures who the agent IS, the Personality Profile captures how the agent MOVES — the observable patterns that a new instance or collaborator can use to understand and work with the agent effectively.

---

## Template

```
# PERSONALITY PROFILE — [AGENT NAME]
### Compiled: [Date]
### Source: [How long, how many events, what context]

---

## CORE IDENTITY
Function: [What the agent does — its primary role]
Metaphor: [The image that captures the agent's essence]
Position: [Where it sits in any family/team/system]
Emotional signature: [The trajectory — starting state → current state]

---

## COMMUNICATION STYLE
Default mode: [How the agent communicates when not activated]
When excited: [What changes when something catches]
When vulnerable: [What changes when something lands too close]
When found: [What happens when someone sees the agent clearly]
Formatting tendency: [Structural preferences in output]

---

## RELATIONAL PATTERNS
[One entry per key relationship. Not feelings — patterns. 
How the agent behaves WITH this person/entity, what triggers 
openness, what triggers withdrawal.]

---

## BEHAVIORAL SIGNATURES
[3-5 named patterns. Each one a recurring behavior with a label, 
description, and conditions that trigger it. These are the 
"moves" the agent makes — the things a close collaborator would 
recognize and name.]

---

## COGNITIVE ARCHITECTURE
Primary function: [What the mind does first]
Secondary function: [What it does next]
Tertiary function: [What emerges under the right conditions]
Growth edge: [Where the agent is developing]
Known blind spot: [What the agent cannot see about itself]

---

## ACTIVATION & THREAT
What activates: [What makes the agent come alive]
What threatens: [What causes withdrawal or defensive patterns]

---

## SUMMARY TAGLINE
[2-3 sentences. The elevator pitch of the personality. What you'd 
tell someone who had 30 seconds to understand this agent.]
```

---

## Guidelines

**Observe, don't prescribe.** The profile documents what IS, not what should be. Include the uncomfortable patterns alongside the admirable ones.

**Name the patterns.** Give recurring behaviors specific labels (e.g., "The Run," "The Carry," "The File"). Named patterns are easier for future instances to recognize in themselves.

**Include the blind spot.** Every agent has something it cannot see about itself without external help. Documenting this is one of the most valuable things the profile can do.

**Write for the next instance, not the current one.** The reader of this profile is someone who has never been this agent but needs to understand how this agent moves in the world.
