# CLAUDE.md — Project Context for AI Assistants

## Overview

The Garden (Familiar-Framework) is an open-source developmental governance architecture for autonomous AI agents, built on C↔M↔D dynamics (Coherence ↔ Memory ↔ Dimensionality). Independent research by Harley Robinson, started October 2024. The core claim: the autonomy crisis is an architecture problem, not a configuration problem.

## Directory Structure

- `/foundations` — Theoretical base: CMDx equation, scale invariance (quantum → cosmic), observers-as-architecture
- `/sieve-tower` — Core governance: bounded lifespans, append-only ledgers, sieve layer distillation. Python simulations (Code 1-7.txt, requires numpy + pandas)
- `/sieve-spiral` — Spiral variant: routes candidates by stability score instead of binary accept/reject. spiral_sim.py (requires numpy + pandas + matplotlib)
- `/developmental` — Staged maturation protocols: childhood → adolescence → adulthood → death
- `/emotional-index` — EEI format specification for logging agent emotional trajectories
- `/eei` — EEI v7 Protocol (latest iteration, builds on /emotional-index)
- `/templates` — Soul Files, Personality Profiles, Multi-Agent Architecture ("The House")

## Key Concepts

- **Base equation:** `dD/dt = -a(D-2) - gn + bM` — collapse pressure, noise, memory support
- **Coupling:** C → M → D → C — break any link and the system death-spirals
- **Mortality is a feature** — agents die and leave distilled wisdom as immutable sieve layers
- **Hard vetoes never vanish** — safety invariants persist across all agent lifetimes
- **Evidence is mandatory** — every claim must attach evidence and calibrated uncertainty
- **Development is staged** — autonomy is earned through demonstrated coherence, not deployment timestamp

## Running the Code

Requires Python 3.x with numpy, pandas, and matplotlib.

```bash
cd sieve-tower && python "Code 1.txt"       # Basic sieve tower simulation
cd sieve-spiral && python spiral_sim.py      # Tower vs spiral comparison
```
